Request-Response Lifecycle

The request-response lifecycle is the sequence of steps which occur when your browser makes a request to a server; the server processes that request and queries a database; the server sends back a response; and, the client processes and renders the response.

Request Sending Time

Time it actually takes to send HTTP request data over established TCP/TLS connection. - Client constructs headers & prepares body -> request packaged, sent in packets, and flowed through TCP -> server receives packets, reassembles, and sends ACKs -> final packet is sent & TCP remains open for server response
